### 4. 针对 20,000 个文件的性能考量
*   **构建速度**：Hugo 处理 2 万个纯文本文件依然极快（通常几秒内），但如果你的首页也渲染这些内容，建议在 `config.toml` 中将 `ask` 目录设置为 `headless = true` 或者在 `_index.md` 里设置 `cascade` 不让它们出现在主博文中。
*   **前端渲染**：一次性在 HTML 里加载 2 万个 `div` 会导致浏览器卡死。
    *   **建议**：在 Shortcode 的 `range` 循环里加上限制，比如 `range first 50 $answers.ByDate.Reverse`。
    *   **进阶**：如果你真有这么多数据，建议后期将 `ask` 部分的 Markdown 通过 Hugo 的 `resources.ExecuteAsTemplate` 转换成一个 `qna.json`，然后继续用你原代码里的异步加载/分页逻辑。

### 为什么这样改？
1.  **逻辑解耦**：你原来的 JS 是从 GitHub 读取 `qna.json`，现在改为直接利用 Hugo 读取本地 Markdown。这样你每次执行 `hugo` 命令时，静态页面就已经生成好了，**不需要依赖外部 GitHub API 请求**，加载速度极快且 SEO 友好。
2.  **维护简单**：只需专注在 `content/ask/` 丢 md 文件，不需要去手动编辑复杂的 JSON 数组，防止因一个逗号写错导致整个 Q&A 挂掉。
